HTML:
The First Step of My Web Development Journey

When I first became interested in technology, I had no idea how websites were created. Every time I opened a website, I wondered how different pages, buttons, images, and menus appeared on the screen. That curiosity inspired me to start learning web development. My journey officially began in Class 9, when I learned HTML (HyperText Markup Language) for the first time. Although HTML looked difficult at the beginning, I soon realized that it is the foundation of every website on the internet. Learning HTML completely changed the way I looked at websites and encouraged me to continue my journey in programming.

The first webpage I created was very simple. It only contained a heading, a few paragraphs, and an image. Even though it looked basic, it gave me confidence because it was something I had built myself. Every new HTML tag I learned felt like discovering a new tool. I enjoyed creating headings, paragraphs, links, images, lists, tables, and forms. Every small success motivated me to learn something new the next day.

As I continued practicing, I discovered that HTML is not responsible for making websites beautiful. Instead, HTML provides the structure of a webpage. It tells the browser where the headings, text, images, buttons, navigation bars, and other elements should appear. Once I understood this concept, learning web development became much easier. Later, I started exploring CSS to design webpages and JavaScript to make them interactive.

Like every beginner, I also faced many challenges. Sometimes my code did not work because I forgot to close a tag. Other times I spent hours trying to find a small typing mistake. At first, these problems were frustrating, but I gradually understood that making mistakes is a natural part of learning programming. Instead of giving up, I learned how to debug my code, read error messages carefully, and search for solutions. Every mistake taught me something valuable.
One of the most important lessons I learned was the importance of regular practice. Reading tutorials alone is never enough. The best way to improve is by writing code every day and building small projects. I practiced creating simple webpages, navigation menus, image galleries, contact forms, and personal portfolio pages. Each project helped me understand HTML more deeply and increased my confidence.
As my skills improved, I realized that web development is not only about writing code. It is also about solving problems and creating websites that are useful for people. A good website should be clean, responsive, easy to navigate, and accessible on both desktop and mobile devices. These principles became an important part of my learning journey.
Today, I continue learning new technologies and improving my web development skills. I enjoy building projects because every project teaches me something different. My portfolio website is one of the projects that represents my progress. It reminds me that every experienced developer once started with simple HTML tags, just like I did in Class 9.
For anyone who wants to start learning HTML, my advice is simple: never be afraid of making mistakes. Programming is a skill that improves with patience and consistent practice. Start with the basics, build small projects, and challenge yourself to create something new every week. Don't compare your beginning with someone else's experience. Focus on learning one concept at a time, and celebrate your progress.
HTML is much more than a programming language for creating webpages—it is the first step toward becoming a web developer. Whether your dream is to become a frontend developer, a full-stack developer, or a software engineer, HTML is where the journey begins. I am grateful that I started learning HTML in Class 9 because it opened the door to web development and inspired me to keep learning every day. My journey is still continuing, and I look forward to building even better websites, learning modern technologies, and helping others who are beginning their own programming journey.
